Evidence Map

研究假设对应的证据

这里聚焦两类核心证据：显式空间事实状态，以及由不确定性触发的主动感知。

研究命题支撑来源被 Spatial Agent 吸收的设计仍未解决的空白
H1 · Agent 需要显式空间事实状态 HoloAgent-0、3DLLM-Mem、Habitat 3.0、RoboCasa、EmbodiedBench 把拓扑、对象、局部几何、可达性、遮挡和 affordance 转化为 Agent 可查询状态。 多数方法仍缺少空间事实与任务图、技能 precondition、空间 verifier 的稳定接口。
H2 · 空间不确定性需要主动感知 SayCan、Code as Policies、具身主动感知/next-best-view 思路、RoboCasa、Habitat 3.0 在低置信度、遮挡、关系冲突和可达性未知时触发 inspect、换视角、探测或请求标注。 现有机器人 Agent 往往被动使用观测，缺少把信息增益、动作成本和安全风险合在一起的主动策略。
系统扩展来源Voyager、Agent Lightning、持续 RL、Datasheets for Datasets、Data Cards 和 OWASP LLM 风险用于支撑长期更新、准入和治理，不作为核心研究假设。
Primary Sources

来源列表

  • Voyager：自动课程、技能库、迭代反馈。
  • SayCan：用 affordance grounding 约束语言模型计划。
  • Code as Policies：语言模型生成可执行机器人策略程序。
  • Open X-Embodiment：跨机器人真实轨迹和 RT-X。
  • Octo：开源通用机器人策略和跨本体适配。
  • OpenVLA：开源 VLA 与微调代码。
  • DROID：多场景真实机器人操作数据，用于研究环境与任务多样性。
  • BridgeData V2：跨环境机器人操作数据集，支持语言和目标条件泛化研究。
  • EmbodiedBench：多模态具身 Agent 长程任务评估。
  • Habitat 3.0：人机协作、导航和重排仿真。
  • RoboCasa：家居厨房操作与 generalist robot benchmark。
  • Agent Lightning：把 Agent 轨迹转为 RL 训练接口。
  • Datasheets for Datasets：记录数据来源、组成、采集过程、用途和限制。
  • Data Cards：面向责任 AI 的结构化数据文档与生命周期透明度。
  • LeRobot：机器人数据采集、训练、控制和评估工具。
  • Isaac Lab：GPU 加速机器人学习仿真与策略训练框架。
  • ROS 2 Managed Nodes：机器人组件生命周期管理和在线替换机制。
  • LangGraph：durable execution、memory、human-in-the-loop。
  • Microsoft Agent Framework：多 Agent workflow、状态和 telemetry。
  • OMEGA：生命周期、契约和可观测性表达方式。
  • 物理世界机器人 Agent 框架：interaction-first、world-model-first、safety-first。
  • OWASP LLM01：prompt injection 风险。
