Month 10-12
Implementation Workstreams

落地不是线性开发，而是四条资产并行成熟

每个阶段都必须同时推进 runtime、spatial task suite、active learning queue 和 release governance，否则系统只能演示，不能持续迭代。

Runtime asset 空间任务图执行器、技能注册表、空间事实状态接口、治理门控、spatial verifier、trace store 和回滚机制。
Evaluation asset 长程任务集、空间谓词标签、遮挡/可达性 holdout、旧空间事实回归集、安全集、unnecessary query 校准集。
Learning asset 空间失败归因、主动感知样本筛选、memory/perception candidate、离线训练和 shadow evaluation。
Deployment asset 权限策略、人工接管、版本回滚、灰度发布、审计日志、指标面板和事故复盘流程。
Resource Envelope

按小团队可执行规模设计

这里不是预算清单，而是研究计划的约束条件：如果资源低于这个范围，应缩小任务族；如果资源更充足，也不应跳过 trace、验证和准入。

人员配置 3-5 人可启动：runtime/后端、机器人控制、感知与空间状态、实验评估、产品交互/安全治理可由多人兼任。
平台配置 一个主仿真环境、一套受控真实空间、一台移动操作平台或可替代的导航底盘 + 机械臂；真实执行先 shadow 和低风险任务。
数据规模 阶段 0-1 先收 100-300 条可回放 episode，覆盖 3-5 个任务族；阶段 2 后扩展主动感知样本和跨域空间 holdout。
审计负载 每周抽样审计空间谓词、主动感知收益、unnecessary query、ask-human 和脱敏状态；高风险 episode 100% 复盘。
First 90 Days

前三个月交付一个可复现实验包

时间工程交付实验交付验收证据
Week 1-3 spatial schema、任务卡、skill contract、trace store、回放脚本。 3 个任务族、每个 2-3 个 seed 场景。 同一 episode 可从 trace 重放空间事实、主动感知、门控、技能结果和 verifier 判断。
Week 4-6 planner、governance、spatial state snapshot、verifier 最小闭环。 text-memory、2D-map、spatial-agent-lite 三个 baseline。 能统计 spatial grounding、relation accuracy、hidden spatial failure 和 human label burden。
Week 7-10 空间失败归因、主动感知策略、数据治理、candidate spatial bundle stub。 100+ 条 episode，覆盖成功、失败、主动感知、人工标注和恢复。 失败能归因到 object、relation、occlusion、reachability、skill、governance 或 data_quality。
Week 11-13 shadow evaluation、量化门槛、报告面板、关键案例回放。 固定 split 的首轮消融实验。 证明 SpatialFact Graph、active perception 和 verifier 是否带来显著增益；若没有，定位失败模块。
Risk Governance

风险治理跟路线一起推进

安全、数据和候选策略不是最后补丁；它们决定每个阶段能否进入真实执行、shadow、灰度或回滚。

动作安全权限、禁区、限速、力控、近人保护和人工确认独立于 planner；低置信度空间事实先 inspect，不直接驱动高风险动作。
数据治理trace 进入训练或候选更新前先检查用途授权、脱敏状态、标签质量、重复失败和安全事件。
候选发布空间事实、感知器或 active policy 候选必须通过旧任务回归、安全集、校准、overblocking 和 shadow run。
策略审计Agentic RL 或恢复策略候选必须证明没有绕过 verifier、滥用 ask-human、制造表面安全或转嫁风险。
Phase Gates

每个阶段必须有可交付物和退出标准

阶段核心产物退出标准不进入下一阶段的条件
0 · 最小空间闭环 spatial schema、runtime skeleton、接口 stub、3-5 个可回放任务 每一步有空间 trace，失败能定位到空间模块边界，并可用同一日志重算指标。 任务只能靠人工脚本串联，无法回放或复现实验。
1 · 空间验证与记忆 spatial world adapter、relation predicates、memory staleness hidden spatial failure 下降，低置信度空间事实会触发 inspect 或 ask-human。 verifier 与最终成功信号高度重复，不能提前发现空间错误。
2 · 主动空间感知 active perception policy、query budget、label queue 空间错误下降，同时 unnecessary query、human burden 和 unsafe event 受控。 主动策略变成过度询问或高成本绕路。
3 · 空间更新 candidate memory/perception bundle、spatial regression suite、shadow report 关系、可达性和 stale detection 提升，旧空间事实和安全集不退化。 新空间事实污染旧任务或绕过安全门控。
4 · 真实灰度 shadow mode、灰度发布、回滚、审计面板 连续运行期间安全事件为 0，主要失败可解释并进入下一轮迭代。 无法稳定回滚、审计缺失或用户接管无法及时生效。
