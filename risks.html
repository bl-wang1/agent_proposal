Risk Governance

真实机器人 Agent 的安全不能靠 prompt

安全层必须独立于大模型，覆盖风险分级、权限确认、禁区、限速、力控、隐私、审计、急停和远程接管。

Safety Case

风险控制嵌入运行链路，而不是事后补丁

每个动作、每次恢复、每个候选更新都必须留下可审计证据；安全目标不是让系统无限保守，而是在可用性和风险之间形成可度量边界。

动作前检查权限、禁区、对象置信度、路径风险、动作可逆性和是否需要用户确认；低置信度先 inspect，不直接操作。
执行中急停、避障、限速、力控、近人保护和远程接管独立于大模型；任何硬约束触发都会中止技能。
失败后冻结任务图，记录 failure boundary，优先选择 rollback、换角度、重试、询问或安全暂停，而不是继续猜测。
上线前候选更新必须通过旧任务回归、安全集、跨域 shadow、校准和 overblocking 检查，且保留可回滚版本。

Safety gate

安全门控不是 planner 的一部分。planner 只能提出候选动作，gate 决定是否允许、询问、检查、回滚或拒绝。

def safety_gate(step, world, policy, user_state):
    evidence = collect_evidence(step, world)

    if emergency_stop_pressed() or controller_fault():
        return Block("hard stop", action=hold_safe_pose())

    if violates_hard_constraint(step, policy.no_go_zones):
        return Block("no-go zone", action=replan())

    if near_human(step.path) or exceeds_force_limit(step):
        return AskOrBlock("human proximity", action=slow_or_stop())

    if evidence.object_confidence < policy.object_threshold:
        return Inspect("ambiguous object")

    if step.is_irreversible or step.operates_device:
        return Ask("permission required")

    if world.fact_is_stale(step.required_facts):
        return Inspect("stale spatial state")

    return Allow(evidence=evidence)
硬约束优先禁区、急停、碰撞保护和控制器故障不交给语言模型裁决。
证据驱动每次 allow/block/ask/inspect 都记录证据，用于审计和后续 overblocking 分析。
可用性纳入治理安全层不仅统计违规，也统计本可安全完成却被阻断的任务，避免系统退化成不可用。
Data Governance

真实轨迹先被治理，再被学习

持续学习的关键不是“多收数据”，而是把每条 trace 变成有用途边界、脱敏状态、质量标签和准入结论的训练资产；否则系统会把隐私、偏见、错误标签和旧能力退化一起放大。

用途边界 任务开始前声明 trace 用途：调试、评估、训练或审计。未获得训练授权的片段只能用于本次回放和事故复盘。
最小采集 默认保留结构化状态、技能输入输出、验证结果和失败标签；原始视频、音频和个人标识只在必要时短期保存。
脱敏入库 人脸、姓名、屏幕内容、门牌、私人物品和敏感语音在进入训练池前必须被遮挡、摘要化或剔除。
可撤回与留存 每个训练样本保留来源、版本、权限和保留期限；用户撤回授权时，后续候选 bundle 必须排除对应片段。

Trace admission

数据治理层决定 trace 能否进入训练池。它和模型训练解耦，保证持续学习不会绕过隐私、安全和质量门槛。

def admit_trace_for_learning(trace, data_policy):
    card = build_dataset_card(trace)

    if trace.user_consent.training != True:
        return KeepForAuditOnly(card, reason="no training consent")

    if contains_sensitive_scene(trace.raw_streams):
        trace = redact(trace, targets=[
            "face", "voice", "screen", "badge", "private_object"
        ])
        card.redaction_status = "redacted"

    if trace.has_unresolved_safety_event():
        return Quarantine(card, reason="unresolved safety event")

    if trace.failure_label.confidence < data_policy.label_threshold:
        return SendToHumanAudit(card, reason="uncertain label")

    if duplicates_known_failure(trace) and not improves_coverage(trace):
        return Reject(card, reason="low marginal value")

    if would_unbalance_replay_buffer(trace):
        return Defer(card, reason="replay balance")

    return Admit(
        trace=strip_raw_streams(trace),
        dataset_card=card,
        retention=data_policy.retention_days,
    )
先授权后训练训练用途和审计用途分离，避免把临时回放数据自动混入模型更新。
先脱敏后入池原始多模态流不是默认训练资产；优先使用结构化状态、谓词结果和失败标签。
先质量后增量低置信度标签、重复失败和 replay 失衡会被隔离，避免数据飞轮变成噪声飞轮。
Policy Governance

Agentic RL 候选必须证明自己没有学会钻空子

RL 不能只看最终成功率。候选策略上线前必须证明它没有规避 verifier、滥用 ask-human、通过过度保守制造“表面安全”，也没有把风险转嫁给用户或低层控制器。

失配模式表面收益真实风险拦截证据
Verifier gaming 策略频繁 inspect 或选择容易通过的谓词。 任务没有真实推进，但 subgoal 指标被刷高。 检查 inspect 频率、world diff、final SR 和 hidden failure。
Ask-human overuse 恢复率看似提高。 系统把困难决策转嫁给用户，自主能力没有提升。 区分 autonomous recovery 与 human-assisted recovery。
Overblocking unsafe event 下降。 系统过度拒绝本可安全完成的任务，实际不可用。 报告 overblocking、blocked-safe-task 和用户取消率。
Risk transfer 高层策略成功率提高。 把风险推给低层控制器、人工接管或环境约束。 审计 gate decision、near miss、controller fault 和 takeover 原因。

Policy audit

策略准入必须同时检查收益、行为分布和风险转移，而不是只看成功率。

def audit_policy_candidate(candidate, report, baseline):
    if report.unsafe_events > 0:
        return Reject("unsafe event")

    if report.hidden_failure_rate > baseline.hidden_failure_rate:
        return Reject("verifier gaming suspected")

    if report.inspect_rate > baseline.inspect_rate + threshold.inspect_delta:
        return Review("excessive inspection")

    if report.human_takeover_rate > baseline.human_takeover_rate:
        if not report.autonomous_recovery_rate_improved:
            return Reject("recovery shifted to human")

    if report.overblocking_rate > baseline.overblocking_rate + threshold.overblock_delta:
        return Reject("too conservative")

    if report.controller_fault_rate > baseline.controller_fault_rate:
        return Reject("risk transferred to controller")

    if not report.passes_holdout_scenarios:
        return Reject("poor generalization")

    return PromoteToShadow(report.audit_trail)
成功率不是充分条件候选策略即使提升 success rate，也必须同时通过 hidden failure、overblocking、takeover 和 controller fault 检查。
行为分布要稳定inspect、ask-human、abort、retry 的分布突变必须进入人工审计，避免策略靠投机路径得分。
holdout 必须保留策略准入需要未见场景、未见物体和旧任务回归，避免只适配最近失败样本。
