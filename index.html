Embodied Spatial Agent Framework

做一个能理解、感知并主动更新空间的机器人 Agent 框架。

ECAOS 把真实环境编码为空间世界模型，让 Agent 用拓扑、对象关系、可达性、遮挡、affordance 和置信度来规划、执行、验证和主动补充感知。

一句话目标

让机器人从“看见物体”走向“理解可行动的空间”。

研究对象面向物理世界长程任务的机器人 Agent。
核心机制空间事实图、主动感知策略、空间验证器、技能契约。
验收方式空间 grounding、关系判断、可达性预测、遮挡处理和长程任务成功率提升。
Architecture Head Figure

ECAOS 的整体架构由四条信息流组成。

Observe 负责把目标和传感器观测写成空间事实；Decide 基于 SpatialFact Graph 选择主动感知或规划；Execute 通过 Skill Contract 调度物理技能；Verify / Learn 把验证结果和 trace 作为候选更新回写。

Observe
Task + Sensors 用户目标、RGB-D/video、机器人位姿、地图、工具反馈、人类标注。
Spatial Perception 对象、区域、局部几何、遮挡、可达性候选和 affordance 候选。
SpatialFact Graph visible、reachable、on/inside/near、confidence、evidence、expires_at、risk_relevance。
Decide
SpatialFact Graph planner 和主动感知策略的共享状态。
Active Perception Policy 低置信、冲突、遮挡、可达性未知或高风险时触发 inspect、换视角、probe 或 ask_human。
Spatial Planner 把目标拆成带 precondition、effect 和 verifier 的空间任务图。
Execute
Spatial Task Graph 子目标、空间谓词、风险等级和恢复边界。
Skill Contract Runtime 按 Contract 调度导航、抓取、放置、开门、VLA policy 或脚本 controller。
Physical World 机器人动作改变真实空间；动作结果重新进入观测和验证。
Verify / Learn
Physical Result 真实动作结果、传感器回读和执行状态。
Spatial Verifier 检查 visible / reachable / on / inside / near 是否成立，发现 hidden spatial failure。
Trace + Candidate Update 记录事实、主动感知、技能调用、门控决策和验证结果；只生成候选更新。
candidate SpatialFacts / perception updates / active policy samples 通过 replay、shadow evaluation 和准入后，回写 SpatialFact Graph。
Executive Overview

研究范围、技术路线与验证协议。

研究目标是把空间信息编码、主动空间感知、任务图规划和空间谓词验证统一为机器人 Agent 框架。

Scope

空间世界模型驱动的移动操作 Agent 框架

服务于可建模、可验证、可主动感知的物理任务；环境差异通过空间编码、感知适配器和技能契约吸收。

Problem 长程失败主要来自空间关系误判、遮挡未处理、可达性估计错误、世界状态过期和技能前提失配。
Approach 以空间事实图、主动感知策略、空间验证器和技能契约构成闭环；trace（执行证据链）用于更新空间事实和感知器。
Benchmark 构建 10-20 个长程任务；每个任务包含 3-8 个子目标，并提供初始状态、成功判据、失败标签和跨域测试集。
Metrics 评估空间 grounding、关系判断、可达性预测、stale memory 检测、主动感知收益、长程成功率和安全事件。
Safety 高风险动作需经显式确认；急停、避障、限速、力控和禁区约束独立于大模型决策链路。
Active Learning 当空间事实低置信度、互相冲突或遮挡严重时，系统主动选择换视角、inspect、低风险探测或请求标注。
Generalization 框架泛化的是空间 schema、谓词、主动感知策略、技能契约和验证逻辑；传感器、地图和控制器按场景替换。
Four Mechanisms

最小研究对象收敛为四个机制。

第一版不追求全功能机器人平台，而是证明空间事实如何被编码、主动补齐、验证并接入技能执行。

SpatialFact Graph把空间写成可查询事实：subject、predicate、object、frame、confidence、evidence、expires_at、risk_relevance。最小谓词先做 visible、reachable、relation(on/inside/near)。
Active Perception Policy当事实低置信度、互相冲突、被遮挡、可达性未知或影响高风险动作时，选择 inspect、change_viewpoint、probe_reachability、ask_human 或 defer。
Spatial Verifier不相信裸 skill success，而是检查 visible、reachable、on/inside/near、path_clear、object_moved 等空间 effect 是否真的成立。
Skill Contract每个技能声明 precondition、effect、verifier、rollback、risk 和 failure code，例如 `place(cup, sink)` 需要 cup in gripper、sink reachable，并产出 inside(cup, sink)。
Research Questions

研究问题也压缩到四个可验证问题。

Fact SpatialFact Graph 能否比文本记忆、2D map 或静态 3D semantic memory 更准确地表达 visible、reachable 和 on/inside/near。
Sense Active Perception Policy 能否用更少 inspect、换视角、低风险探测和人工标注降低空间错误。
Verify Spatial Verifier 能否比技能返回值更早发现 hidden spatial failure。
Act Skill Contract 能否把不同导航、操作和 VLA 技能统一接入 precondition/effect/verifier，而不污染任务图。
