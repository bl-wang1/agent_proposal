Architecture Head Figure

整体架构：四条信息流

Observe 写入空间事实；Decide 由 Planner + Gate 生成候选任务图并决定是否放行；Execute 由 Skill Runtime 调用具体技能；Verify / Learn 验证 effect 并回写 trace。

01 Input goal, constraints, RGB-D, robot pose, skill feedback
02 Observe objects, relations, occlusion, reachability candidates
03 SpatialFact Graph confidence, evidence, conflict_set, expires_at, risk
04 Decide: Planner task nodes with preconditions, effects and recovery edges
05 Gate allow, inspect, ask-human, block, rollback
06 Execute: Skill Runtime navigate, grasp, place, open, VLA/controller wrapper
07 Spatial Verifier checks visible, reachable, inside/on/near against effect
08 Trace + Update failure boundary, candidate facts, replay, shadow admission

Executive Overview

研究范围、技术路线、验证协议

目标：把空间事实、主动感知、任务图和空间谓词验证统一到机器人 Agent runtime。

Scope

主动空间感知机器人 Agent 框架

服务于可建模、可验证、可主动感知的物理任务；环境差异通过空间编码、感知适配器和技能契约吸收。

Problem 长程失败主要来自空间关系误判、遮挡未处理、可达性估计错误、世界状态过期和技能前提失配。
Approach 空间事实图、主动感知策略、空间谓词验证器构成闭环；技能契约和 trace 作为支撑接口。
Benchmark 10-20 个长程任务；每个任务 3-8 个子目标，覆盖初始状态、成功判据、失败标签和跨域测试。
Metrics 评估空间 grounding、关系判断、可达性预测、stale memory 检测、主动感知收益、长程成功率和安全事件。
Safety 高风险动作需经显式确认；急停、避障、限速、力控和禁区约束独立于大模型决策链路。
Active Learning 当空间事实低置信度、互相冲突或遮挡严重时，系统主动选择换视角、inspect、低风险探测或请求标注。
Generalization 泛化空间 schema、谓词、主动感知、技能契约和验证逻辑；替换传感器、地图和控制器。
Three Contributions

三个核心贡献

聚焦三件事：空间事实状态、基于不确定性的主动感知、空间谓词 verifier。

SpatialFact Graph把空间写成可查询事实：subject、predicate、object、frame、confidence、evidence、expires_at、risk_relevance。优先做必要的证据融合、冲突处理和过期门控。
Uncertainty-Guided Active Perception当事实低置信度、互相冲突、被遮挡、可达性未知或影响高风险动作时，按信息收益、任务价值、动作成本和风险选择 inspect、change_viewpoint、probe_reachability、ask_human 或 defer。
Spatial Predicate Verifier不相信裸 skill success，而是用多视角视觉/深度、几何状态一致性或人工/仿真 oracle 评估 visible、reachable、on/inside/near 等 effect 是否成立。
Research Questions

三个研究问题

Fact SpatialFact Graph 能否比文本记忆、2D map 或静态 3D semantic memory 更准确地表达 visible、reachable 和 on/inside/near。
Sense Active Perception Policy 能否用更少 inspect、换视角、低风险探测和人工标注降低空间错误。
Verify Spatial Verifier 能否比技能返回值更早发现 hidden spatial failure。
System Layers

六层系统

系统分成六个职责层：交互接收目标，空间状态保存事实，Agent 推理生成计划，技能执行动作，安全治理做门控，评估运维负责回放和发布。

01人机交互目标输入、任务卡、确认、打断、接管。
02空间事实状态地图、物体、人、关系、时间、置信度。
03Agent 推理目标理解、任务图、技能选择、失败解释。
04技能执行感知、导航、操作、沟通、恢复。
05安全治理权限、禁区、限速、力控、隐私、审计。
06评估运维回放、指标、失败分类、灰度发布。
Runtime Flow

一条运行主线

一次任务沿着目标、状态、动作、验证和更新前进；验证结果和 trace 再回写为空间事实，驱动下一轮决策。

1Goal用户给任务，系统展示计划和风险。
2State空间事实图提供当前假设。
3Actruntime 调度带契约的技能。
4Verify空间谓词验证器判断 expected relation/effect 是否达成。
5Updatetrace 形成空间证据链，用于更新 SpatialFact、感知器和主动学习队列。
feedback更新后的事实、置信度和失败标签回到 State。
