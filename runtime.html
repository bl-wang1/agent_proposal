Runtime Loop

一次任务如何闭环执行

图像与指令 -> Observe -> SpatialFact -> Active Sense -> 任务图 -> 技能执行 -> 谓词验证 -> trace 更新；运行页只跟踪这一条 episode 路径。

Episode Walkthrough

示例流：从图像和指令到 trace

运行闭环页跟踪一个具体任务，不再重复系统总图；每个模块都展示该步收到什么、产出什么，以及什么时候触发主动感知。

失败则补证据后重试
01 Input 图像 + 文本指令 视觉观察和用户目标进入 episode。

示例指令：“把桌上的杯子放进水槽，然后擦桌面。”

02 Observe 抽取空间候选 从观测中抽对象、区域、关系和遮挡。

输出候选：cup, table, sink；on(cup, table)；occluded_by(edge)；pose confidence 0.62。

03 SpatialFact 结构化空间事实 候选写成带证据和置信度的状态。
visible(cup)=0.71 reachable(cup)=? inside(cup,sink)=false
04 Active Sense 主动补证据 不确定、冲突或风险触发感知动作。

reachable 未知且杯沿遮挡 -> change_viewpoint + probe_reachability。

05 Planner 输出可验证任务图 补证据后生成候选任务节点。
localizegraspplaceverifywipe

每个节点带 precondition、effect、verifier 和 fallback。

06 Gate + Skill 门控后执行 动作先过 gate，再调用具体技能。

allow grasp(cup)；dispatch pick policy；失败则进入 recover branch。

07 Verify 空间谓词验证 用 effect 谓词判断任务是否推进。

独立检查 inside(cup, sink)；裸 skill success 不算任务完成。

08 Trace 写入证据链 记录输入、决策、执行、验证和恢复。

input、SpatialFact diff、active sense、task graph、skill I/O、verifier result。

每一步都有输入和输出

Input -> Observe输入是图像流、机器人位姿和文本目标；输出是对象、区域、关系、遮挡和可达性候选。
Observe -> SpatialFact候选被写成带 confidence、evidence、frame、expires_at 的空间事实，而不是自由文本描述。
SpatialFact -> Active Sense低置信、冲突、遮挡或高风险前置条件触发 inspect、换视角、probe 或 ask_human。
Active Sense -> Planner补证据后再生成任务图；Planner 输出的是候选节点，不直接拥有执行权。
Planner -> Verify -> Trace技能执行后检查空间谓词 effect；成功、失败、恢复和人工选择都写入 trace。
Verify -> Active Sense如果 effect 未成立，例如 inside(cup, sink) 失败，回到主动感知补视角或探测，再 replan/retry。
Episode Evidence Chain

Trace：一次任务的证据链

trace 串起意图、证据、风险、结果和恢复动作，用来复盘任务是否真的推进。

阶段写入 trace 的证据失败时如何处理后续用途
Goal accepted 用户目标、任务卡、对象引用、约束、确认状态和数据用途授权。 目标或对象歧义时进入 clarify，不生成物理动作。 用于复盘目标理解错误和 ask-human 策略。
Step planned task graph node、precondition、expected effect、verifier、rollback 和风险等级。 前置条件缺失时重新规划或 inspect。 用于训练 planner、progress checker 和任务模板。
Gate decided allow/ask/inspect/block 决策、触发规则、世界证据、用户选择。 高风险或低置信度时询问、检查、回滚或终止。 用于分析 overblocking、安全违规和权限策略。
Skill executed skill implementation、输入输出、控制器状态、耗时、传感器摘要。 失败定位到 skill_failure 或 implementation_mismatch。 用于 skill success predictor、adapter routing 和 contract test。
Progress checked predicate result、confidence、world diff、hidden failure 标记。 effect 未达成时触发 diagnose 和 recovery policy。 用于训练 verifier，并计算 subgoal SR 与 hidden failure rate。
Learning admitted failure label、脱敏状态、dataset card、回归结果、shadow report。 低质量、未授权或高风险片段隔离，不进入训练池。 用于候选 bundle、发布门槛和事故复盘。

Runtime main loop

主循环只检查一件事：下一步是否推进任务；否则恢复、求助或停止。

def run_episode(goal, constraints):
    observation = sensors.read_rgbd_and_pose()
    spatial_facts = observe.write_facts(observation, goal)

    while spatial_facts.has_unresolved_uncertainty():
        sense_action = active_sense.choose(spatial_facts)
        new_observation = runtime.dispatch(sense_action)
        spatial_facts = observe.update_facts(new_observation)

    ctx = assemble_context(
        goal=goal,
        constraints=constraints,
        world=spatial_facts.snapshot(),
        skill_stats=registry.stats(),
        failure_memory=trace_store.retrieve(goal),
    )

    task_graph = planner.plan(ctx)
    trace = Trace(goal, task_graph, ctx.world)

    while not task_graph.done():
        step = task_graph.next_ready_step()
        skill = registry.select(step.intent, ctx.world)

        decision = governance.check(
            skill=skill,
            precondition=skill.precondition,
            risk=skill.risk_level,
            human_context=ctx.human_state,
        )

        if decision.requires_human:
            approval = interaction.ask_for_approval(step, decision.reason)
            trace.record_human_decision(step, decision, approval)
            if not approval.granted:
                trace.mark_aborted(decision.reason)
                break

        if not decision.allowed:
            recovery = planner.replan(task_graph, reason=decision.reason)
            if decision.requires_inspect:
                recovery = active_sense.choose(ctx.world, reason=decision.reason)
            trace.record_gate_block(step, decision, recovery)
            continue

        result = runtime.dispatch(skill, step.args, timeout=skill.timeout)
        progress = verifier.check(step.expected_effect, observe.write_facts(sensors.read_rgbd_and_pose()))
        trace.record(step, skill, result, progress)

        if progress.ok:
            task_graph.mark_done(step)
        else:
            failure = diagnose(step, result, progress)
            recovery = recovery_policy.choose(failure, ctx)
            task_graph.apply_recovery(recovery)

    trace_store.write(trace)
    learning_queue.enqueue_if_useful(trace)
governance.check风险、权限、禁区、限速和人工确认放在 planner 外部。
独立 verifier技能 success 不等于任务 progress；effect 由状态谓词检查。
learning queue真实轨迹先诊断、筛选，再进入候选更新。
